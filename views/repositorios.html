Repositorios

¿Qué es un repositorio?

Los repositorios son instalaciones virtuales donde se depositan los resultados de la investigación científica o de un trabajo realizado en un proyecto.

Un repositorio de Git es un almacenamiento virtual de tu proyecto. Te permite guardar versiones del código a las que puedes acceder cuando lo necesites.

¿Cuales son los repositorios mas comunes?

Al trabajar en repositorios Git hay unaamplia variedad de repositorios sobre los cuales se puede trabajar, aunque normalmente cada empresa o proyecto manejan sus propios, sin embargo a dia de hoy puedes trabajar sobre algunos libres.

GitHub: Este sin duda es el mas conocido de todos, ya que ahi se almacena la mayoria de proyectos tanto grandes como pequeños, Kubernetes o Docker tienen su data montada ahi para que la gente pueda usarla, los proyectos normalmente se almacenan ahi como es el caso del semillero de Fedesoft;
Azure: Algunas empresas como siigo manejan sus repositorios montados en ev Azure ya que facilita mucho la revision de Pull request y es un poco mas privado que GitHub, de tal manera que se maneja en un entorno mas empresarial y mas completo.

¿Como funciona un repositorio?

Git funciona del siguiente modo:

1: Crea un "repositorio" (proyecto) con una herramienta de alojamiento de Git (por ejemplo, Github);
2: Crea un "repositorio" (proyecto) con una herramienta de alojamiento de Git (por ejemplo, Bitbucket)
3: Añade un archivo a tu repositorio local y confirma ("commit") los cambios
4: Envía ("push") los cambios a la rama principal
5: Haz cambios en tu archivo con una herramienta de alojamiento de Git y confírmalos
6: Extrae ("pull") los cambios a tu máquina local
7: Crea una rama ("branch", versión), haz algún cambio y confírmalo
8: Abre una solicitud de incorporación de cambios ("pull request": propón cambios a la rama principal)
9: Fusiona ("merge") tu rama con la rama principal

Git

Git es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia, la confiabilidad y compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente. Su propósito es llevar registro de los cambios en archivos de computadora incluyendo coordinar el trabajo que varias personas realizan sobre archivos compartidos en un repositorio de código.

Instrucciones mas comunes de Git

Hay una serie de ordenes, comandos o instrucciones basicas en Git, las cuales son las siguientes:

git init: Esto crea un subdirectorio nuevo llamado .git, el cual contiene todos los archivos necesarios del repositorio – un esqueleto de un repositorio de Git. Todavía no hay nada en tu proyecto que esté bajo seguimiento.
git fetch: Descarga los cambios realizados en el repositorio remoto.
git merge nombre_rama: Impacta en la rama en la que te encuentras parado, los cambios realizados en la rama “nombre_rama”.
git commit -m "comentario" Confirma los cambios realizados. El “mensaje” generalmente se usa para asociar al commit una breve descripción de los cambios realizados.
git push origin nombre_rama: Sube la rama “nombre_rama” al servidor remoto
git status: Muestra el estado actual de la rama, como los cambios que hay sin commitear.
git add: Comienza a trackear el archivo “nombre_archivo”.
git branch -a: Lista todas las ramas locales y remotas.
